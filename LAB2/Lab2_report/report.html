    LAB 2 | Interfacing with the Web

    Introduction:
    In this lab we connected our Raspberry Pi to the internet and enabled it send live data from our sensors to a database. Create a dashboard that displays all the sensor data in a running chart in a user friendly web app. One can start by connecting the Pi to the database, send sensor information to the database, and lastly create an online dashboard to display all the data.

  • Step 1: Firebase Realtime Database
    - This NoSQL database is constantly looking for new sensor readings from a RPi.
    - Instal Python library called pyrebase that takes care of the communication between python and Firebase.
    - Create a write_to_pyrebase Python function that takes one parameter per sensor (temperature, humidity, light, gate, envelope, audio) and pushes those reading to Firebase everytime it runs.
  • Step 2: Cron
    - Is the Linux timing- utility tool that schedules tasks and will be used to run the python script.
    - It is imperative to use absolute paths, because crontab works best with absolute paths.
    - Create a main.py file that reads data from the sensors and passes them to the write_to_pyrebase python function.
    - Using cron, schedule the main.py script to run once for a minute for every minute.
  • Step 3: WebApp and Firebase
    - Design a Pyramid web app that enables the user to see the live sensor data in action.
    - Create the app that is able to interact with Firebase.
    - Use the Google charts API to plot and visualize the data obtained from firebase.
    - Write a Javascript file that takes care of the live-updates for the graphs.
